1. Overview

JFEM is a linear static finite element solver written in Julia that reads MSC Nastran BDF (Bulk Data File) input decks and produces results compatible with Nastran SOL 101. It is designed for educational use and as a validation tool for structural analysis workflows.

Key features:

  • Reads standard fixed-format and free-format Nastran BDF files
  • Solves linear static analysis (equivalent to SOL 101)
  • Exports results as VTK files (for ParaView visualization) and JSON
  • Includes a validation tool for comparing results against Nastran F06 output

2. Requirements

  • Julia 1.9+ (tested on Julia 1.10 and 1.11)
  • Required Julia packages:
    • LinearAlgebra (stdlib)
    • SparseArrays (stdlib)
    • Statistics (stdlib)
    • Printf (stdlib)
    • JSON
    • WriteVTK

3. Installation

  1. Clone or download the repository
  2. Install required Julia packages: 
    julia -e 'using Pkg; Pkg.add(["JSON", "WriteVTK"])'
  3. Verify the installation: 
    cd JFEM/src
julia main.jl
    This should print the usage message.

4. Quick Start

To solve a Nastran BDF model:

julia src/main.jl path/to/model.bdf

This will:

  1. Parse the BDF file
  2. Assemble and solve the stiffness system
  3. Write a VTK file (model_Subcase_N.vtu) for ParaView visualization
  4. Write a JSON results file (model.JU.JSON)

Output files are placed in a FEM_output/ directory next to the BDF file.

To compare with Nastran results:

# First, convert the Nastran F06 to JSON:
julia src/f06_2_json.jl path/to/model.f06

# Then compare:
julia src/compare_results.jl model_name path/to/FEM_output

5.2 Converting F06 to JSON

julia src/f06_2_json.jl <model.f06> [output_dir]

Parses a Nastran F06 output file and writes <model>.NAST.json with the same JSON structure as the solver output. This is required for the comparison tool.

6.4 Output Format

The JSON output contains:

  • displacements — Translational (t1, t2, t3) and rotational (r1, r2, r3) DOFs per grid
  • spc_forces — Reaction forces at constrained grids
  • forces — Element forces:
    • CQUAD4/CTRIA3: fx, fy, fxy (membrane), mx, my, mxy (bending), qx, qy (transverse shear)
    • CBAR: axial, shear_1, shear_2, torque, moment_a1, moment_a2, moment_b1, moment_b2
  • stresses — Element stresses at fiber distances z1 (bottom) and z2 (top)
  • strains — Element strains at fiber distances z1 and z2

7. Theory & Implementation

7.1 CQUAD4 Element

The CQUAD4 implementation uses a Mindlin-Reissner formulation with the following features:

Element Coordinate System

The element x-axis bisects the angle between the two diagonals G1–G3 and G2–G4, following the Nastran convention. Specifically:

  • The element normal ze is computed from the cross product of the diagonals
  • The x-axis direction is: x_raw = normalize(d13) - normalize(d24)
  • This is then projected onto the element plane for the final xe
  • ye = ze × xe

Membrane Behavior

Standard isoparametric bilinear shape functions with incompatible modes (Wilson-Taylor bubble functions) to eliminate parasitic shear locking in membrane response. The incompatible mode DOFs are eliminated by static condensation at the element level.

Bending Behavior

The bending stiffness uses the standard Mindlin plate formulation with the PSHELL 12I/T³ scaling parameter (default = 1.0). A value of 4.0 in the PSHELL card means the effective bending stiffness is 4× the standard plate value.

Transverse Shear

The MITC4 (Mixed Interpolation of Tensorial Components) formulation is used to prevent transverse shear locking. Covariant shear strains are sampled at tying points on the element edges and interpolated to the Gauss points.

Drill Rotation

A Hughes-Brezzi drill rotation stabilization is applied with a small stiffness proportional to the element area to provide a non-singular DOF 6 (rotation about normal).

Stress Recovery

Gauss-point averaged strains (including incompatible mode contributions) are used for stress recovery. The incompatible mode amplitudes are recovered via static condensation: α = -Kbb-1 Kba u.

7.2 CTRIA3 Element

Constant strain triangle (CST) membrane element. The strain field is constant across the element (no bending capability). A small drill rotation stiffness is added for compatibility with shell assemblies.

7.3 CBAR Element

The CBAR uses a 3D Timoshenko beam formulation with 6 DOFs per node (3 translations + 3 rotations).

Shear Deformation

For PBARL ROD sections, the Cowper shear correction factor is used:

κ = 6(1+ν) / (7+6ν)

The shear area is As = κ · A. Setting the shear factor to zero gives Euler-Bernoulli (no shear deformation) behavior.

Local Coordinate System

The bar x-axis is from node GA to node GB. The y-axis is determined from the orientation vector V specified on the CBAR card: z = x × V, y = z × x.

Force Recovery

Element forces are recovered from f = Ke ue in local coordinates with Nastran sign conventions:

  • Axial = flocal[7] (tension positive)
  • Shear_1 = -flocal[2] (plane 1, xy-plane)
  • Shear_2 = -flocal[3] (plane 2, xz-plane)
  • Torque = -flocal[4]
  • Moments at ends A and B follow Nastran equilibrium: Mb1 - Ma1 + V1L = 0

7.4 RBE3 Element

RBE3 is an interpolation element (not a rigid body element). It distributes forces applied at the reference grid to the dependent grids using a least-squares approach:

  1. Forces are distributed equally among dependent grids
  2. Moments are distributed as force couples using the cross-product matrix and least-squares
  3. Displacements at the reference grid are interpolated as the weighted average of dependent grid displacements, including rigid body offset corrections
Note: RBE3 elements do not add stiffness to the model. They only affect load distribution and displacement interpolation.

7.5 SNORM (Surface Normal Averaging)

The SNORM parameter (default = 20°) controls the averaging of surface normals at shell nodes shared by multiple elements. For each node:

  • If all connected element normals are within the SNORM angle of the averaged normal, the node uses the averaged normal as its local z-axis
  • At fold lines (angle exceeding SNORM), the original coordinate system is preserved

This affects the transformation of displacements between global and local coordinates.

9. Known Limitations

  • PBARL sections: Only the ROD cross-section type is implemented. Other types (BOX, I, T, etc.) are not supported.
  • CTRIA3 bending: The CTRIA3 is a membrane-only element. No bending or transverse shear capability.
  • CQUAD4 transverse shear recovery: Transverse shear forces (qx, qy) are output as zero. The MITC4 formulation handles shear in the stiffness but the force recovery for qx/qy is not implemented.
  • Coordinate systems: Only CORD2R (rectangular) is supported. CORD2C (cylindrical) and CORD2S (spherical) are not.
  • Solution types: Only SOL 101 (linear static). No eigenvalue, frequency response, or nonlinear analysis.
  • GRAV loads: Gravity loads are detected but not implemented (a warning is printed).
  • Large models: The direct sparse solver may be slow for models with more than ~50,000 DOFs. No iterative solver option is available.
  • RBE2: Not supported. Only RBE3 (interpolation) is implemented.
  • CBEAM: Not supported. Only CBAR.
  • Thermal loads: Not supported.
  • PBAR (non-library): Not supported. Use PBARL with ROD type.
